# run-seo-evaluation

## Inputs
- `alternate_name`
- `start_date`, `end_date`
- `variant_urls`: pre-enriched list from `resolve-deal-urls` (orchestrator pre-resolved these from `dim_deal`; the upstream URL resolver / Okta / MDS-insights stages stay off the path).
- `out_path`: where to write `seo_<alt>.json`
- `passthrough_dir`: where to write the upstream HTML/XLSX

This skill no longer accepts a control URL set. Synthetic-control DiD is generated by upstream `seo-impact-plugin:seo-impact-analyzer` (hierarchical L3→L2→L1→page_type→domain fallback).

## What this skill does

0. **Pre-flight: cache freshness.** The cached plugin can be stale even when `marketplace.json` pins a recent commit — Claude Code keys its cache on `plugin.json:version`, not commit SHA, so upstream pushes that don't bump the version are silently ignored. **Before invoking any upstream skill**, grep the cached `impact_analyzer.py` for `compute_verdict`:
   ```powershell
   Select-String -Path "$env:USERPROFILE\.claude\plugins\cache\<marketplace>\seo-impact-plugin\<version>\scripts\impact_analyzer.py" -Pattern "compute_verdict"
   ```
   If no hit, the cache predates the verdict feature. Abort with `{"status":"failed","reason":"stale plugin cache — run: rm -rf ~/.claude/plugins/cache/<marketplace>/seo-impact-plugin && /plugin marketplace update <marketplace>","failed_at":"cache-precheck"}` and surface the fix command to the user. Do NOT proceed with the run — it would silently produce `verdict=null`.

1. **Tag input.** Tag every variant URL with `group="variant"`. Drop any URLs flagged as MDS or non-`groupon.com` host. Verify the pre-enriched URL list is non-empty.
2. **Guardrails.** Dispatch `seo-impact-plugin:seo-guardrails` with `release_date=start_date`. Capture `signal_level` (`early` | `partial` | `full`) or abort reason.
   - If aborted → write `{"status":"skipped","reason":...}` to `out_path`, return.
   - If passed → CONTINUE.
3. **Page classification.** Dispatch `seo-impact-plugin:seo-page-classifier` with the variant URL list.
4. **GSC fetch.** Dispatch `seo-impact-plugin:seo-gsc-fetcher` with `release_date=start_date` and the classified URLs.
5. **Analysis.** Dispatch `seo-impact-plugin:seo-impact-analyzer`. Output is `output/impact_analysis_<run_id>.json`. The analyzer computes synthetic-control DiD, power analysis, verdict, and per-category aggregates internally — do NOT post-process them.
6. **Passthrough rendering.** Dispatch `seo-impact-plugin:seo-report-generator`. Writes the upstream self-contained HTML to `output/seo_impact_report_<run_id>.html` (and an XLSX). Copy both into `<passthrough_dir>/`:
   - `<passthrough_dir>/seo_<alt_name>.html`
   - `<passthrough_dir>/seo_<alt_name>.xlsx`
7. **Read upstream JSON + HTML.**
   ```python
   import base64, json
   from pathlib import Path
   ia = json.loads(Path("output/impact_analysis_<run_id>.json").read_text(encoding="utf-8"))
   html_b64 = base64.b64encode(
       Path("<passthrough_dir>/seo_<alt_name>.html").read_bytes()
   ).decode("ascii")
   ```
8. **Write JSON intermediate** to `out_path`:
   ```python
   out = {
       "status": "ok",
       "alternate_name": "<alt_name>",
       "verdict": ia.get("verdict"),
       "did_coherence": ia.get("did_coherence"),
       "signal_level": ia.get("meta", {}).get("signal_level") or "<from guardrails>",
       "did": ia.get("did") or {},
       "power_analysis": ia.get("power_analysis") or {},
       "overall": ia.get("overall") or [],
       "summary_tables": ia.get("summary_tables") or {},
       "by_category_l1": ia.get("by_category_l1") or [],
       "by_category_l2": ia.get("by_category_l2") or [],
       "by_category_l3": ia.get("by_category_l3") or [],
       "by_page_type": ia.get("by_page_type") or [],
       "caveats": ia.get("caveats") or [],
       "upstream_html_b64": html_b64,
       "passthrough_html": "passthrough/seo_<alt_name>.html",
       "passthrough_xlsx": "passthrough/seo_<alt_name>.xlsx",
   }
   Path("<out_path>").write_text(json.dumps(out), encoding="utf-8")
   ```
   Do NOT embed `url_details` from the upstream JSON — it can be 18k+ rows and bloats the combined report. Full URL detail lives in the upstream XLSX passthrough.

## Tool contract
- Never invoke `seo-impact-plugin:okta-proxy`, `seo-impact-plugin:seo-url-resolver`, or `seo-impact-plugin:seo-mds-insights-review`. The orchestrator pre-enriches URLs; these stages add Okta dependencies we don't want on the critical path.
- `bq` CLI only for any direct BQ access.

## Failure modes
- Guardrails abort → graceful skip with reason in JSON.
- Empty variant URL list → `{"status":"no_urls"}`.
- Any delegated upstream skill fails → retry once, then write `{"status":"failed","reason":...,"failed_at":"<step>"}`.
- Upstream impact_analysis JSON missing `verdict` (older upstream commit) → write `{"status":"failed","reason":"upstream missing verdict — pin marketplace.json to commit 3100dc8 or later","failed_at":"impact-analyzer"}`.
